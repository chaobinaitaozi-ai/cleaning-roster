# 掃除当番アプリ デプロイ（配信）手順書

このアプリをブラウザ版として社内で共有するための手順です。

## 1. アプリをビルドする

まずはブラウザで動く形式にファイルをまとめます。

```bash
npm run build
```

実行後、プロジェクト内に `dist` というフォルダが作成されます。この中身が配信に必要なすべてのファイルです。

## 2. 配信（デプロイ）方法の選択

社内の環境に合わせて以下のいずれかの方法を選んでください。

### 方法A: 社内のWebサーバーに置く
既存の Apache, Nginx, または Windows Server (IIS) などがある場合、`dist` フォルダの中身をサーバーの公開ディレクトリにコピーするだけで完了です。
- 例: `\\internal-server\www\cleaning-app\`

### 方法B: GitHub Pages / Vercel / Netlify を使う
クラウドサービスを利用可能な場合、最も簡単に無料で配信できます。
- **Vercel**: GitHubと連携するだけで、URL（自動生成）が発行されます。
- **GitHub Pages**: リポジトリの Settings > Pages から、`dist` フォルダ（または `main` ブランチ）を指定して公開できます。

### 方法C: 共有ドライブで簡易共有 (非推奨だが必要な場合)
Webサーバーを立てるのが難しい場合、`dist`フォルダを共有ドライブに置き、`index.html`をダブルクリックして開くことも可能ですが、ブラウザのセキュリティ設定（CORS等）により動作しない場合があります。基本的には方法AまたはBを推奨します。

## 3. データの共有方法

このアプリは現在「各ユーザーのブラウザ」にデータを保存します。

- **初期設定**: 管理者が「担当者マスタ」で全員を登録し、「エクスポート」から `cleaning_staff_list.json` を保存します。
- **共有**: 上記のJSONファイルを社内チャット（Teams/Slack等）や共有フォルダで配布します。
- **利用開始**: 各メンバーは、アプリを開いて「インポート」からそのJSONファイルを読み込むことで、同じ名簿・同じ当番サイクルで利用を開始できます。
